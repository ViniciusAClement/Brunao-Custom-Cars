 Task List: Integração Frontend-Backend para Brunao Custom Cars

## Requisitos Gerais
- Integrar o frontend React com o backend Spring Boot via APIs REST.
- Implementar autenticação JWT para sessões seguras.
- Gerenciar roles: apenas usuários logados como GERENTE podem criar products, clients e funcionarios.
- Substituir estado local por dados do backend (H2/PostgreSQL).
- Adicionar validações, tratamento de erros e feedback ao usuário.
- Usar Axios para chamadas HTTP.
- Gerenciar estado global com Context API ou Redux.
- Implementar navegação com React Router para login/protegido.
- Garantir responsividade e acessibilidade básica.

## Requisitos Funcionais
- Tela de login para autenticação com email/senha.
- Logout que invalida o token.
- Dashboard protegido com abas para CRUD de products, clients e funcionarios.
- Apenas GERENTE pode criar/editar/deletar recursos.
- Listagem de dados do backend.
- Formulários com validação (ex.: email único, CPF).
- Mensagens de sucesso/erro para operações.

## Dependências e Configuração
- Instalar Axios, React Router, e opcionalmente Redux Toolkit.
- Configurar proxy no Vite para o backend (localhost:8080).
- Definir base URL da API em constantes.

## Passos de Implementação

### 1. Configuração do Projeto e Dependências
- Instalar dependências: `npm install axios react-router-dom @reduxjs/toolkit react-redux` (ou similar para estado).
- Configurar Vite proxy em `vite.config.js` para redirecionar `/api` para `http://localhost:8080`.
- Criar arquivo de constantes para URLs da API (ex.: `const API_BASE = '/api'`).
- Adicionar variáveis de ambiente (.env) para configurar a URL base da API.
- Testar instalação com `npm run dev`.

### 2. Implementar Autenticação e Gerenciamento de Estado
- Criar Context/AuthContext para gerenciar token JWT, user info e role.
- Implementar funções de login/logout com Axios (POST /auth/login, POST /auth/logout).
- Armazenar token em localStorage/sessionStorage.
- Criar hook `useAuth` para acessar contexto.
- Adicionar interceptor Axios para incluir token em headers automaticamente.
- Adicionar refresh de token automático em caso de expiração.
- Testar contexto com componente de teste.

### 3. Navegação e Proteção de Rotas
- Configurar React Router com rotas: `/login`, `/dashboard` (protegida).
- Criar componente `ProtectedRoute` que verifica token/role antes de renderizar.
- Redirecionar para login se não autenticado ou sem role GERENTE para ações específicas.
- Criar layout com navbar/menu lateral.
- Adicionar botão de logout na navbar.
- Testar fluxo de navegação e redirecionamento.

### 4. Substituir CRUD Local por APIs Backend - Products

- POST /products: criar produto (apenas GERENTE, validar antes de enviar).
- PUT /products/{id}: editar produto (apenas GERENTE).
- DELETE /products/{id}: deletar produto (apenas GERENTE).
- Substituir `useState` local por `useEffect` para buscar dados na montagem.
- Atualizar lista após operações.
- Implementar loading states e tratamento de erros.

### 5. Substituir CRUD Local por APIs Backend - Clients

### 6. Substituir CRUD Local por APIs Backend - Funcionarios

## Considerações Adicionais
- O backend já tem um gerente padrão (gerente@empresa.com / senha123).
- Testar com Postman primeiro para validar endpoints.
- Migrar dados locais para backend se necessário.
- Conformidade com LGPD para dados pessoais.
- Considerar usar TypeScript para melhor type-safety.
- Estrutura de pastas sugerida: components/, pages/, services/, contexts/, constants/.

## Riscos e Mitigações
- Exposição de token: armazenar seguro, usar HTTPS.
- Ataques CSRF: usar JWT stateless.
- Dados inconsistentes: sincronizar estado com backend.
- Performance: lazy loading para listagens grandes.
- XSS: sanitizar inputs e outputs.
- Segurança: validar no backend também, não confiar apenas em validações frontend.

## Tecnologias Utilizadas
- Frontend: React 19.2.4, TypeScript, Vite, React Router, Axios.
- Backend: Spring Boot 3.2.0, Spring Security, JWT (JJWT), JPA/Hibernate.
- Database: H2 (desenvolvimento), PostgreSQL (produção).
- HTTP Client: Axios.
- State Management: Context API (simples) ou Redux Toolkit (complexo).
- UI Notifications: react-toastify ou similar.

## Endpoints Backend Disponíveis
- POST /auth/login - Login com email/password.
- POST /auth/logout - Logout e invalidar token.
- POST /auth/register - Registrar novo usuário (apenas GERENTE).
- GET /products - Listar produtos.
- POST /products - Criar produto (apenas GERENTE).
- PUT /products/{id} - Editar produto (apenas GERENTE).
- DELETE /products/{id} - Deletar produto (apenas GERENTE).
- GET /clients - Listar clientes.
- POST /clients - Criar cliente (apenas GERENTE).
- GET /funcionarios - Listar funcionários.
- POST /funcionarios - Criar funcionário (apenas GERENTE).

## Glossário
- JWT: JSON Web Token, formato seguro para autenticação stateless.
- GERENTE: Role/função de usuário com permissão de criar/editar recursos.
- Context API: API do React para gerenciar estado global.
- Axios: Cliente HTTP para fazer requisições.
- React Router: Biblioteca para navegação em SPAs.
- Interceptor: Função que intercepta requisições/respostas do Axios.
- CORS: Cross-Origin Resource Sharing, política de segurança para requisições cross-domain.
